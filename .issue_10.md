## Goal

Wire the session and agent-cursor overlay tools from `cua-driver` into the `computer` skill as `computer_*` MCP tools. This closes the session/cursor gaps from #4 and provides the desktop agent with the ability to manage multi-agent concurrent session contexts and configure color-coded visual cursor overlays.

The tools to wire are:
- Sessions: `start_session`, `end_session`
- Cursor Overlay: `set_agent_cursor_enabled`, `set_agent_cursor_style`, `set_agent_cursor_motion`, `get_agent_cursor_state`

---

## Background — what's available (verified)

`cua-driver` daemon exposes six tools related to sessions and agent-cursor-overlays. In the new architecture, these should be mapped through `mcp_server.py`.

---

## Scope

1. **MCP Server Integration (`code/mcp_server.py`)**:
   Expose 6 new MCP tools:
   - `computer_start_session(session: str)`
   - `computer_end_session(session: str)`
   - `computer_set_agent_cursor_enabled(enabled: bool, cursor_id: str = "")`
   - `computer_set_agent_cursor_style(...)`
   - `computer_set_agent_cursor_motion(...)`
   - `computer_get_agent_cursor_state()`
   - Route all arguments to `_cua("start_session", ...)` etc.

2. **Skill Tool Catalog (`code/skills.py`)**:
   Add detailed JSON schemas for each of the 6 tools into `_TOOL_CATALOG`.

3. **Skill Registration (`code/agent_config.yaml`)**:
   Add the 6 tools under `tools_allowed` of the `computer` skill config.

4. **Smoke Tests (`code/tests/test_computer_mcp_tools.py`)**:
   Add mock subprocess execution tests for each tool to verify command payload construction.
