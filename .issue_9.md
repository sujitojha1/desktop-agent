## Goal

Wire the `start_recording`, `stop_recording`, `get_recording_state`, `replay_trajectory`, and `install_ffmpeg` tools from `cua-driver` into the `computer` skill as `computer_*` MCP tools. This closes the recording/replay gaps from #4 and provides the desktop agent with the ability to record trajectories for debugging and execute deterministic regression testing.

---

## Background — what's available (verified)

`cua-driver` daemon exposes five tools related to trajectory recording, replaying, and recording dependencies:
* `start_recording` (output_dir, record_video)
* `stop_recording`
* `get_recording_state`
* `replay_trajectory` (dir, delay_ms, stop_on_error)
* `install_ffmpeg` (confirm)

These tools persist recording configurations inside the daemon. In the new architecture, these should be mapped through `mcp_server.py`.

---

## Scope

1. **MCP Server Integration (`code/mcp_server.py`)**:
   Expose 5 new MCP tools:
   - `computer_start_recording(output_dir: str, record_video: bool = False)`
   - `computer_stop_recording()`
   - `computer_get_recording_state()`
   - `computer_replay_trajectory(dir: str, delay_ms: int = 500, stop_on_error: bool = True)`
   - `computer_install_ffmpeg(confirm: bool = False)`
   - Route all arguments to `_cua("start_recording", ...)` etc.

2. **Skill Tool Catalog (`code/skills.py`)**:
   Add detailed JSON schemas for each of the 5 tools into `_TOOL_CATALOG`.

3. **Skill Registration (`code/agent_config.yaml`)**:
   Add the 5 tools under `tools_allowed` of the `computer` skill config.

4. **Smoke Tests (`code/tests/test_computer_mcp_tools.py`)**:
   Add mock subprocess execution tests for each tool to verify command payload construction.
