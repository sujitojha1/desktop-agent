## Goal

Wire the remaining input, window, and diagnostic driver tools from `cua-driver` as `computer_*` MCP functions (via `cua-driver call` through the daemon) to achieve full tool parity. This closes the remaining gaps from #4 and provides the desktop agent with rich gestures and window control.

The tools to wire are:
- Input/Gestures: `double_click`, `right_click`, `drag`, `move_cursor`, `get_cursor_position`
- Window/App Management: `bring_to_front`, `kill_app`
- Diagnostics/Perception: `debug_window_info`, `zoom`

---

## Background — what's available (verified)

`cua-driver` daemon is running and exposes these 9 commands. They accept a variety of arguments (such as `element_index`, coordinates `x`/`y`, modifier keys, and drag endpoints), which must be correctly mapped to MCP schemas and validated.
