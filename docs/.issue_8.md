## Goal

Wire the `page` tool from `cua-driver` (allowing CDP/WebKit DOM interaction for Electron, Chromium, and Tauri apps) as `computer_page` under the `computer` skill. This closes the tool gaps from #4 and provides the desktop agent with the ability to query the DOM, execute JavaScript, click elements via CSS selectors, and extract text from web/Electron content.

---

## Background — what's available (verified)

`cua-driver` daemon exposes the `page` tool. It enables interacting with target browsers/Electron windows (such as VS Code, Cursor, Slack, Notion) that run on debugging ports. The tool supports actions like `execute_javascript`, `get_text`, `query_dom`, `click_element`, and `enable_javascript_apple_events`.

---

## Scope

1. **MCP Server Integration (`code/mcp_server.py`)**:
   Expose the `computer_page` MCP tool.
   - Arguments: `action`, `pid`, `window_id`, `selector`, `css_selector`, `javascript`, `attributes`, `bundle_id`, `user_has_confirmed_enabling`.
   - Forward variables cleanly to `_cua("page", args)`.

2. **Skill Tool Catalog (`code/skills.py`)**:
   Register `computer_page` in `_TOOL_CATALOG` with detailed schemas matching the `cua-driver describe page` definition.

3. **Skill Registration (`code/agent_config.yaml`)**:
   Add `computer_page` under the `tools_allowed` list of the `computer` skill.

4. **Smoke Tests (`code/tests/test_computer_mcp_tools.py`)**:
   Add unit smoke tests validating CLI argument construction under mock subprocess execution.
