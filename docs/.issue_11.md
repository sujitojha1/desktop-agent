## Goal

Wire the config and introspection tools from `cua-driver` into the `computer` skill as `computer_*` MCP tools. This enables the desktop agent to retrieve and set daemon configuration parameters, check system permission status, and verify if driver updates are available.

The tools to wire are:
- Persistent Config: `get_config`, `set_config`
- Introspection / Diagnostics: `check_permissions`, `check_for_update`

---

## Background — what's available (verified)

`cua-driver` daemon exposes these 4 tools. They should be exposed through `mcp_server.py` as MCP tools and wired into the `computer` skill.

---

## Scope

1. **MCP Server Integration (`code/mcp_server.py`)**:
   Expose 4 new MCP tools:
   - `computer_get_config()`
   - `computer_set_config(...)`
   - `computer_check_permissions()`
   - `computer_check_for_update()`
   - Route all arguments to `_cua("get_config", ...)` etc.

2. **Skill Tool Catalog (`code/skills.py`)**:
   Add detailed JSON schemas for each of the 4 tools into `_TOOL_CATALOG`.

3. **Skill Registration (`code/agent_config.yaml`)**:
   Add the 4 tools under `tools_allowed` of the `computer` skill config.

4. **Smoke Tests (`code/tests/test_computer_mcp_tools.py`)**:
   Add mock subprocess execution tests for each tool to verify command payload construction.
